Onboarding a Team Member (Mark or Nik)
They run — one command
./setup.sh --instance=[firstname] --domain=infra-pe
What setup.sh does automatically
  1. Installs GitHub CLI (gh) via Homebrew if missing
  2. Runs gh auth login if not authenticated
  3. Creates Private-[Name]-Dashboard and Private-[Name]-Config under their GitHub account
  4. Wires their local config dir to the new config repo and pushes first commit
  5. Installs Python deps, Google OAuth, API keys, LaunchAgents
You do — one command after they finish
gh repo add-collaborator ygontownik/Read-Tomac-Deal-Pipeline <their-github-handle> --permission read
Gives them read-only access to the deal pipeline. They cannot push.

How to use

Call check_all() from _auth_watchdog at the top of any pipeline script to gate auth-dependent sections:

from _auth_watchdog import check_all
health = check_all()                               # writes auth_health.json
if health['jefferies']['status'] == 'failed':
    print('Jefferies auth expired — skipping'); skip_jefferies = True

Dashboard auto-refreshes every 60 s while this tab is open. Watchdog writes ~/credentials/auth_health.json; this tile reads it.
