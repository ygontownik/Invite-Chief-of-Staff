1
Install prerequisites
10 minutes  ·  one-time

Python 3.11+ and Git. Run these in Terminal to check:

Terminal
python3 --version git --version

If Python is missing or below 3.11:

https://www.python.org/downloads/

If Git is missing: run xcode-select --install in Terminal.

Homebrew (brew.sh) makes macOS package installs easier if you don't have it yet.

GitHub account. Needed later to host your private firm config repo so teammates can pull shared settings. Free at:

https://github.com/signup

Claude Code CLI — required only for SKILL-based pipelines (personal briefing, podcast transcription). Not required for email triage, dashboard, or transcript processing.

Two modes — pick one:
API only — Anthropic API key is enough. Email triage, call transcript memos, action items, dashboard all work. No Claude Code subscription needed. Monthly cost: ~$15–30.
API + Claude Code — Adds personal daily briefing and podcast transcription via scheduled SKILL pipelines. Requires Claude Code CLI installed and authenticated (~$20/mo additional). Recommended for full functionality.

If you want the full experience, install Claude Code now (requires Node.js 18+):

Terminal
node --version # need 18+; install from nodejs.org if missing npm install -g @anthropic-ai/claude-code claude login # opens browser — sign in with your Anthropic account
If you're starting with API-only mode, skip the Claude Code install. You can add it later without redoing any other steps.
2
Create an Anthropic account and get an API key
5 minutes  ·  one-time per firm

The pipeline uses Claude (Anthropic's AI) to classify emails, extract action items, and write memos. You need your own account and API key.

a
Sign up and add a credit card (pay-per-use, no monthly fee until you use it):
https://console.anthropic.com
b
Go to API Keys → Create Key → name it cos-pipeline
c
Copy the key (starts with sk-ant-) — you'll paste it during setup
Keep this key private. Don't put it in any file — the setup script stores it in macOS Keychain where scripts can access it without it ever touching disk.
3
Create Google Cloud OAuth credentials
10 minutes  ·  most complex step

The pipeline reads your Gmail and writes to Google Docs. Google requires you to create OAuth credentials to authorize this. It's free — you're creating a "private app" for your own use.

1
Create a new project and name it cos-pipeline:
https://console.cloud.google.com
2
With that project selected, go to APIs & Services → Library and enable these three APIs one at a time:
  • Gmail API
  • Google Drive API
  • Google Docs API
3
Go to APIs & Services → OAuth consent screen
  • User type: External
  • App name: COS Pipeline, support email: your email
  • Click through to Test users → add your email address
  • Save and continue through remaining screens
4
Go to APIs & Services → Credentials → Create Credentials → OAuth 2.0 Client ID
  • Application type: Desktop app
  • Name: cos-pipeline-desktop
  • Click Create
5
Click Download JSON on the credential that was just created. Save it as:
Terminal
mkdir -p ~/credentials mv ~/Downloads/client_secret_*.json ~/credentials/gdrive_credentials.json
If you get lost anywhere in Google Cloud Console, search for the page name in the top search bar — it works well.
4
Clone the pipeline and run setup
5 minutes  ·  installs everything

Clone the pipeline code (public repo — no credentials needed) and run the interactive setup script:

Terminal
git clone https://github.com/ygontownik/Dashboard.git ~/cos-pipeline cd ~/cos-pipeline ./setup.sh

The setup script will ask you for:

  • Your name and role — used in AI prompts and memo attribution
  • Your firm name and short name — e.g. "Cascade Capital Partners" / "CCP"
  • Email provider — gmail or outlook
  • Anthropic API key — from Step 2
  • Dashboard username and password — anything you want; used for local HTTP auth

It also opens a browser for Google OAuth sign-in (uses the credentials file from Step 3). Sign in with your Google account and click Allow.

5
Customize your firm identity and investment focus
15 minutes  ·  edit one file

Rather than editing the YAML directly, use the configuration wizard — it asks you questions and generates the correct config blocks to paste in:

→ Open the configuration wizard

Or open the file directly if you prefer editing YAML:

Terminal
open -a TextEdit ~/cos-pipeline/firm_context.yaml

Shared firm sections (everyone on your team uses these):

  • investment_focus — your sector priorities in order (e.g. "power & utilities", "digital infrastructure")
  • team — your partners/colleagues with their roles; drives speaker attribution on call transcripts
  • peer_firms — competitors and co-investors you want flagged in intel
  • key_people — counterparties and contacts you want flagged in action items
  • deal_keywords in firm_config.json — active deal names, target company names

Personal section — goes at the bottom of your local copy only. Never commit this to the shared repo. It controls your personal briefing and podcast feeds:

firm_context.yaml — personal: block
personal: briefing_email: "you@yourdomain.com" # where daily briefing is emailed podcast_feeds: - show: "Catalyst" rss: "https://feeds.megaphone.fm/catalyst" - show: "Your Favorite Show" rss: "https://example.com/feed.rss" briefing_time: "07:00" # local time for daily briefing delivery

Each person on the team adds their own personal: block to their local firm_context.yaml with their own email and podcast feeds. If you share a config repo, this section stays out of it — add it after pulling.

Keychain service prefix. In firm_config.json, set "keychain_service_prefix" to something unique for your firm (e.g. "cascade-pipeline"). This isolates your secrets from any other firm's installation on the same machine and makes your API billing fully self-contained.
6
Create your output Google Docs
5 minutes  ·  auto-created

The pipeline writes to four Google Docs: Follow-ups, Deal Pipeline, People/CRM, and Recruiting. Run the auto-create command and it will make blank Docs in your Drive and populate firm_config.json with their IDs:

Terminal
cd ~/cos-pipeline python3 setup.py --create-docs

This opens a browser once (Google OAuth). After that, you'll see the four new Docs appear in your Google Drive. Their IDs are saved automatically — no manual copying needed.

You can also create the Docs manually and paste their IDs into firm_config.json under the "docs" key if you prefer to name them yourself.
7
Set up podcast transcription (optional)
5 minutes  ·  skip if not using podcasts

If you added podcast feeds to your personal: block in Step 5, the pipeline transcribes each new episode using AssemblyAI and writes structured memos to Google Docs. You need your own AssemblyAI account — Anthropic is not used for this step.

1
Sign up (free tier available; transcription billed per minute after that):
https://www.assemblyai.com
2
Go to your dashboard → copy your API Key
3
Store it in your Keychain (same prefix as your other secrets):
Terminal
security add-generic-password \ -s "cos-pipeline/ASSEMBLYAI_API_KEY" \ -a "$USER" -w "YOUR_KEY_HERE" -U
Replace cos-pipeline with your keychain_service_prefix if you changed it.
Typical cost: ~$0.009/minute of audio. A 45-minute podcast episode costs about $0.40. Most users spend $5–15/month on this.
8
Connect your call transcript source
5 minutes  ·  add Drive folder IDs

The pipeline can process call transcripts that land in Google Drive — from Otter AI, Zoom, or any other tool that saves to Drive. Add your folder IDs to firm_context.yaml under transcript_sources:

firm_context.yaml
transcript_sources: - type: "google_drive_folder" name: "Otter AI" folder_ids: - "YOUR_FOLDER_ID_HERE" # paste from the Drive URL category_hint: "auto"

To get a folder ID: open the folder in Google Drive, copy the string at the end of the URL after /folders/.

Skip this if you don't use Drive-based transcripts. The pipeline works without it — email triage and the dashboard run independently.
9
Validate and install background tasks
2 minutes

Check everything is wired up correctly:

Terminal
cd ~/cos-pipeline && python3 setup.py --validate

Install the three background tasks (if setup.sh didn't do this already):

Terminal
cd ~/cos-pipeline && ./setup_launchagents.sh all

Then open the dashboard:

Terminal
open http://localhost:7777

Your firm is set up.

Three tasks are now running automatically on your machine.

Adding colleagues to your firm's pipeline

Once you're set up, adding team members is faster — they don't repeat the Google Cloud or Anthropic steps. You do two things once, they do one thing:

  1. Create a private GitHub repo for your firm config (firm_context.yaml, firm_config.json, drive-docs.yaml). Push your configured files there. This becomes your shared source of truth.
  2. Add their GitHub account as a collaborator on that private repo.
  3. Send them the team onboarding link — they clone both repos, store the API key you give them, and run OAuth with their own Google account.

Create your private config repo:

Terminal
# Create a new private repo on GitHub, then: mkdir ~/my-firm-config cp ~/cos-pipeline/firm_context.yaml ~/my-firm-config/ cp ~/cos-pipeline/firm_config.json ~/my-firm-config/ cd ~/my-firm-config git init && git remote add origin https://github.com/YOUR_ORG/YOUR-config.git git add . && git commit -m "Initial firm config" git push -u origin main

Then set COS_CONFIG_DIR="$HOME/my-firm-config" in your ~/.zshrc and tell new team members to clone it there. The pipeline picks it up automatically.

Common questions

Will this touch all my email or just specific things?
It reads recent email using OAuth (revocable at any time from your Google account settings). It only processes emails that match your configured keywords or senders. Everything else is skipped. Nothing is sent — it only reads and writes to your own Google Docs.
Can I run this on two machines (office + home)?
Yes. Install on each machine. The dedup system checks the shared Google Docs before writing — if machine A already processed something, machine B skips it. You don't need to coordinate.
How do I update when new pipeline features are released?
Terminal
git -C ~/cos-pipeline pull # gets pipeline code updates git -C ~/my-firm-config pull # gets any config changes you pushed from another machine
How do I stop the background tasks?
Terminal
cd ~/cos-pipeline && ./setup_launchagents.sh --uninstall
Is my firm's data isolated from other firms using this software?
Completely. Your API key, Google credentials, and output Docs are yours. The pipeline code is open source and shared, but it processes only your data against your accounts. No data passes through any intermediary. You can audit every script in ~/cos-pipeline/.
