Set up your pipeline

5 steps  ·  ~50 minutes  ·  done once by one person

Joining an existing team Config repo already exists — your admin sends you a link Setting up for a new firm First install, starting from scratch, you are the admin
1
Configure your identity first
Before you touch a terminal — shapes every memo the pipeline produces
~10 min

START HERE — Open the configuration wizard

The wizard asks you about your investment focus, analytical style, email voice, and active deals. Your answers are woven into every memo, briefing, and action item the pipeline produces.

Open the wizard →

The wizard generates your CLAUDE.md, firm_context.yaml, and firm_config.json. Keep the browser tab open — you'll paste the output into files in Step 3.

What is Claude Code CLI? It's Anthropic's terminal tool that runs AI-powered pipeline scripts on a schedule. Required only for the daily briefing feature — email triage and the dashboard work without it.
2
Create your accounts
API keys you'll paste when the installer asks
~15 min
GitHub account
https://github.com/signup
You'll need your GitHub username in Step 3 when you clone the repo.
Anthropic API key
https://console.anthropic.com/settings/keys
Create a new key, copy it. The installer asks for it and stores it in macOS Keychain — it never touches a file.
AssemblyAI API key (for podcast transcription)
https://www.assemblyai.com/dashboard
Free tier included. Create account, copy the API key from the dashboard. ~$0.009/min billed after the free tier.
Claude Code CLI — optional, for daily briefing
Terminal
npm install -g @anthropic-ai/claude-code claude
Requires Node.js 18+. Run node --version to check. The second command opens authentication in your browser.
Google Cloud OAuth is handled automatically by the installer in Step 3 — you don't need to do anything in the Google Cloud Console manually.
3
Run the installer
Four commands — most of the work happens automatically
~20 min
1 Clone the pipeline code
Terminal
git clone https://github.com/ygontownik/Dashboard.git ~/cos-pipeline
2 Enter the pipeline folder
Terminal
cd ~/cos-pipeline
3 Run setup — installs dependencies, keys, and Google OAuth
Terminal
./setup.sh
  • Installs Python packages (pyyaml, google-auth, anthropic, assemblyai…) ✓ automatic
  • Stores Anthropic and AssemblyAI API keys in macOS Keychain ✓ automatic
  • Opens Google OAuth in your browser — sign in, click Allow ✓ automatic
  • Installs background LaunchAgents (email triage, dashboard) ✓ automatic
4 Create your Google Drive structure automatically
Terminal
python3 setup_new_firm.py
  • Creates all Google Drive folders (Chief of Staff, Transcripts, Deals…) ✓ automatic
  • Creates all six Google Docs (Briefing Log, Follow-ups, Deal Pipeline…) ✓ automatic
  • Writes firm_context.yaml with every folder and doc ID filled in ✓ automatic
  • Writes firm_config.json with Keychain prefix and keyword arrays ✓ automatic
4
Connect your transcript app
Paste two folder IDs — takes two minutes
~2 min

The setup_new_firm.py script printed two folder IDs at the end. Paste them into your transcription app now:

Otter AI

Paste the Otter AI folder ID printed by the setup script.

Settings → Integrations → Google Drive → Connect → paste folder ID

Firefly

Paste the Firefly folder ID printed by the setup script.

Settings → Sync → Google Drive → paste folder ID
From now on, any transcript saved by Otter or Firefly will be picked up by the pipeline automatically within 24 hours. No manual action needed.
5
First run & validate
Two commands — then everything runs automatically
~3 min
Validate everything is wired up
Terminal
python3 ~/cos-pipeline/setup.py --validate
Run the pipeline for the first time (processes last 7 days)
Terminal
python3 ~/cos-pipeline/cos_capture_pipeline.py --since 168h
Open your dashboard
http://localhost:7777
The pipeline runs automatically every day. You only need to run commands manually if you want to process something immediately or test a change.
