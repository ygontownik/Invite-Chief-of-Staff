1
Create a GitHub access token
2 minutes  ·  one-time

The TCIP config repo is private. You need a token to download it. This is a one-time step — you can revoke it later if you want.

  1. Go to github.com/settings/tokens/new
  2. Note: TCIP pipeline
  3. Expiration: 1 year
  4. Scopes: check repo (the top checkbox under that heading)
  5. Scroll down → Generate token
  6. Copy the token that appears (starts with ghp_) — you'll paste it in the next step
Save it now. GitHub only shows the token once. Paste it somewhere safe before moving on.
2
Run the installer
5 minutes  ·  downloads repos, installs dependencies

Open Terminal (search in Spotlight: Terminal) and run these two commands. The second command is interactive — it will ask for your GitHub username, your token (from Step 1), and your API key.

Terminal
curl -fsSL https://ygontownik.github.io/Dashboard/onboard.sh -o /tmp/tcip-onboard.sh bash /tmp/tcip-onboard.sh

The script will walk you through everything — no other commands needed. It tells you what it's doing at each step.

If anything goes wrong, just re-run bash /tmp/tcip-onboard.sh. It's safe to run again.
3
Enter the API key when prompted
1 minute  ·  stored in macOS Keychain

During the install, you'll be prompted for an Anthropic API key. This is TCIP's shared key — get it from Yoni before running the installer. It looks like:

Format
sk-ant-api03-…

The key is stored in your macOS Keychain — not in any file. It never leaves your machine and is not in any code.

You'll also set a dashboard username and password. Use anything — this is just basic auth for the local web interface. Write it down.

4
Sign in to Google when the browser opens
2 minutes  ·  one-time OAuth

The installer will open a browser window asking you to sign into Google. This gives the pipeline read access to your Gmail and write access to the shared TCIP Google Docs.

  1. Sign in with your Google account (the one with access to TCIP docs)
  2. Click Allow on the permissions screen
  3. Return to Terminal — the script continues automatically
If the browser doesn't open automatically: Terminal will print a URL — paste it into Chrome or Safari manually.
5
Place the Google credentials file
2 minutes  ·  file copy

Yoni will share a file called gdrive_credentials.json via a secure channel. This is the Google Cloud OAuth client — it's what allows the pipeline to open the browser sign-in flow.

  1. Save the file Yoni sends you
  2. Move it to exactly this path:
Terminal
mkdir -p ~/credentials mv ~/Downloads/gdrive_credentials.json ~/credentials/gdrive_credentials.json

If setup.sh already ran before this file was in place, just run it again:

Terminal
cd ~/cos-pipeline && ./setup.sh --resume
6
Verify everything is running
1 minute

Run a quick validation to confirm the pipeline can reach all its dependencies:

Terminal
cd ~/cos-pipeline && python3 setup.py --validate

Then open the dashboard:

Terminal
open http://localhost:7777

You should see the TCIP dashboard with live data. From here it runs automatically — no manual steps needed day-to-day.

You're set up.

Common questions

Who pays for the API?
TCIP has one Anthropic account. All three of you (Yoni, Mark, Nik) use the same API key stored in your own Keychains. Charges go to one TCIP bill regardless of whose machine runs a pipeline job. Yoni can see per-model usage in ~/dashboards/logs/anthropic-usage.jsonl.
Will my pipeline duplicate what Yoni already processed?
No. Each script checks the shared Google Docs before writing. If an email thread or transcript is already there (written by any team member's machine), it skips it. Email dedup runs by subject line; transcript dedup runs by a PROCESSED: header written to the source doc.
Where does the TCIP config live?
In ~/cos-pipeline-config/ — cloned from the private GitHub repo. Shared team config (deal keywords, Doc IDs, peer firms) lives there. Your personal podcast feeds go in your local copy only and are never committed.
How do I update the config when Yoni pushes changes?
Terminal
git -C ~/cos-pipeline-config pull
Something broke. How do I debug?
Logs are in ~/dashboards/logs/claude-tasks/. Each pipeline has a .stdout.log and .stderr.log. Send the relevant log to Yoni.
