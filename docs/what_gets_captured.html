What Gets Captured

What's automatic, what you need to save, and what stays out of scope.

What you need to save — call transcripts
📋
Call transcripts
Scanned from your configured Drive folders · processed daily at 7:22am + real-time webhook
Save to Drive
Where to save
Any Google Drive folder listed under transcript_sources in your firm_context.yaml
Supported formats
.txt .vtt .srt .md .rtf Google Docs .m4a .mp3 .pdf
Where results go
Follow-ups Doc (action items) · Pipeline Doc (deal threads) · People Doc (contacts)
How to get transcripts into Drive — by recording service
Service How to connect
Otter AISet up a Zapier automation: New Otter recording → Upload to Google Drive folder. Use the folder ID from your transcript_sources config. One-time setup, then automatic.
Beside AIIn Beside AI settings, enable Google Drive sync and point it at your configured folder.
FirefliesIn Fireflies integrations, connect Google Drive and select your transcript folder.
GrainIn Grain workspace settings, enable Drive export to your configured folder.
Read AIEnable Drive integration in Read AI settings, point at your folder.
FathomIn Fathom settings, enable Drive sync.
Zoom / Teams / MeetIf using the built-in call_recorder.py, recordings save to ~/recordings automatically. Configure a local_folder source pointing there.
ManualPaste transcript text into a .txt file, save to any configured Drive folder. The filename becomes the call title.
Finding your folder IDs

Your configured transcript folders are in firm_context.yaml under transcript_sources:

firm_context.yaml
transcript_sources: - type: "google_drive_folder" name: "Otter AI" folder_ids: - "1zJly0cCiqsbZ3umYBXse7nYE7tUpFGOr" ← this is the folder ID - "1pHmuq_TfLY46GDg0BzRIwrq57ictIT5S"

To open a folder in Drive from its ID:

URL pattern
https://drive.google.com/drive/folders/PASTE_FOLDER_ID_HERE

To find a folder ID from an existing Drive folder: open it in Drive, copy the last segment of the URL.

Adding a new transcript source
firm_context.yaml
transcript_sources: - type: "google_drive_folder" name: "Beside AI" # shown in logs and memo headers folder_ids: - "YOUR_BESIDE_AI_FOLDER_ID" # from your Beside AI Drive sync settings category_hint: "auto" # or "Deal" / "Personal" / "Other"

Then configure the recording service to sync to that folder. No code changes needed.

What does NOT get captured
Content Why it's excluded
Google Drive files outside your configured foldersDrive API only queries the specific folder IDs you listed
Google Docs you didn't create or explicitly shareOAuth scope limits access to designated files only
PDFs (research reports, pitch decks, etc.)Not yet supported — text extraction from PDFs is a planned future feature
Slack / Teams messagesNo integration built
Calendar event content / meeting notesCalendar is read for scheduling context only, not content
Files on your Desktop not in a configured folderNot scanned unless you add a local_folder source pointing there
Other people's emailsPipeline reads your inbox only, authenticated with your credentials

How the pipeline decides what to process
1
List files
For each folder_id in transcript_sources, lists files modified since the last run. On --backfill, processes all files regardless of modification date.
2
Check dedup
Skips files already in ~/credentials/processed_cos_transcripts.json.
3
Filter format
Skips audio files and unsupported formats (PDFs, images, spreadsheets).
4
Process
Pass 1 (Sonnet memo) → Pass 2 (Opus extraction) → writes action items and deal intel to shared Google Docs.
5
Mark done
Adds file ID to the dedup tracker so it won't be processed again.
Re-process a specific file
Terminal
python3 cos_otter_backfill.py --force --id FILE_ID
Check what would be processed (dry run)
Terminal
python3 cos_otter_backfill.py --list
